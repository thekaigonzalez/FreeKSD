LIBMEMIO - Library for interacting with low level memory access functions

    LIBMEMIO Allows functions to be passed in Lua and allows memory editing in a time.

FUNCTIONS

    Functions are:

    AllocateMemory(mem) [1]
    Assert(expr) [2]
    AllowUniversalLua() [3]
    MemPrint(memset, str) [4]

INIT SYNOPSIS

    int luaopen_libmemio(lua_State* L);

FUNCTION SYNOPSIS

    int alloclua(lua_State *L);

    int assertflua(lua_State *L);

    int allowUluafile(lua_State* L);

    int fastprint(lua_State *L);

WARNING


    This library is only for Quality of Life and is special
    to the Kux framework. When using LIBMEMIO in production,
    You may encounter certain bugs such as MAVs, NULL Terminations,
    And Slower applications. Please refer to EXAMPLES(1) for proper use of functionalities.

EXAMPLE

    --- Low level KuxAPI memory functions
    require('libmemio')

    AllowUniversalLua() -- Allows Unsafe Memory handling to be passed through our Lua Program

    AllocateMemory(1) -- Allocates "1" byte of memory

    MemPrint(10, "Hello World!") -- Allocates "10" bytes of memory for the string and frees it afterwards.

    Assert(LuaExpression expr) -- Asserts the expression just like the C Function. Please refer to https://man7.org/linux/man-pages/man3/assert.3.html for the Official C Documentation of assert().
